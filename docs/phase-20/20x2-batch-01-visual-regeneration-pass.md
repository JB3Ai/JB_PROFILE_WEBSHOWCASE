## Generation Path Options

| Option | Description | Benefits | Risks | Required controls | Recommendation |
| --- | --- | --- | --- | --- | --- |
| Generate inside workflow using approved 20W5 prompts | A future milestone generates candidate images directly from the approved prompts, then reviews them as private intake candidates. | Most traceable path; prompt, output, and review can be recorded together. | Generated files could accidentally be treated as production assets if controls slip. | Keep generated candidates private, do not optimise, do not export WebP, do not register, do not render, and do not stage raw sources without approval. | Preferred for 20X3. |
| Generate externally and supply candidates as private intake | A human or design/image tool creates candidate visuals externally, then supplies them for private review. | Useful if external tools provide better art direction or iteration speed. | Harder to capture prompt/tool/source metadata; risk of unmanaged rights or embedded text issues. | Record source, tool, prompt, date, rights notes, and review status before any future optimisation. | Acceptable fallback. |
| Supply manually created candidate images for review | Human-created or existing purpose-made candidates are supplied locally for review. | May be faster if candidates already exist. | Rights, source, and public-safety provenance must be checked carefully. | Treat as intake-only; do not move, rename, optimise, publish, link, stage, or commit unless separately approved. | Acceptable only with complete metadata. |
| Split by product | Generate safer products first, or split flagship visuals from rights-sensitive visuals. | Allows OS3 Dash / Super Agent OS to move first while DukeBox or VoiceGrid remains held if needed. | Can create milestone fragmentation and inconsistent batch readiness. | Define exact product subset and preserve the same safety gates for each split. | Optional if all five cannot move together. |
| Hold and revise prompts first | Pause generation and run a prompt revision milestone. | Best if any prompt is unsafe, vague, or off-brand. | Slows visual progress. | Use 20W6 only if human review requests changes. | Not needed unless human review rejects a 20W5 prompt. |

## Candidate Intake Rules

Any generated or manually supplied candidate must follow these rules:

- Private intake only.
- Do not commit raw PNG/JPG files unless explicitly approved.
- Do not move into `public/`.
- Do not optimise.
- Do not export WebP.
- Do not register.
- Do not render.
- Record source, prompt, tool, date, and review status.
- Check for embedded text issues.
- Check for private-data hallucination.
- Check for third-party IP resemblance.
- Check for unsupported claims.
- Check desktop and 390px mobile crop.
- Keep rejected or held candidates out of production.

## Supplied Image Review Path

If supplied images are used, they may be temporarily reviewed only as private intake candidates.

Suggested temporary review location:

`docs/media-intake/06-generated-product-visuals/batch-01/`

This location is for intake review only and must not become a public asset path.

Suggested intake naming pattern:

`<product-slug>-candidate-<source-or-tool>-<yyyymmdd>-v01.<ext>`

Metadata to record before review:

- Product.
- Source or tool.
- Prompt used, if generated.
- Creator or supplier, if known.
- Date supplied.
- Original filename.
- Dimensions.
- File type.
- Rights/source notes.
- Review status.
- Rejection or revision notes, if any.

What must not be staged:

- Raw PNG/JPG candidates.
- Existing source images.
- Generated source images.
- Intake PDFs.
- Videos.
- Any visual file without explicit human approval.

What requires human approval before optimisation:

- Prompt match.
- Negative prompt compliance.
- Source/usage rights.
- Third-party IP resemblance.
- Embedded text.
- Public-safety risks.
- Desktop crop.
- 390px mobile crop.
- Filename and registry key suitability.

GTR3 / IsikoloAI / Ndebele style handling:

GTR3 / IsikoloAI styled images must not be mixed into Batch 01 unless explicitly approved by a separate milestone. They should remain separate campaign or visual-intake candidates.

Ndebele / South African visual language references may inspire future IsikoloAI or campaign-specific work only after a dedicated style/source review. They must not be applied to Batch 01 product visuals without a milestone that approves that direction.
