## Final Image Prompt Rules

All final prompts must follow these shared rules:

- Premium dark-mode visual style.
- JB3Ai / OS3 visual language.
- Slate Blue Steel, Deep Navy-Black, Chrome, Luminous Tech Green.
- Public-safe conceptual design.
- No real private data.
- No real client data.
- No screenshots of private systems.
- No legal, family, court, evidence, or financial material.
- No credentials, API keys, repository secrets, or real dashboards.
- No third-party copyrighted assets.
- Low or no text inside images.
- No unsupported claims embedded in the image.
- Must work inside a product card.
- Must crop cleanly on desktop and 390px mobile.

### A. OS3 Dash

Product positioning:

Founder OS / Operating Dashboard.

Final image prompt:

Create a premium dark-mode conceptual founder command-dashboard visual for OS3 Dash. Show a staged, fictional dashboard made from sharp slate-blue steel panels on a deep navy-black background, with chrome dividers and subtle luminous tech-green signal accents. The composition should communicate structured workflow visibility, daily command, approvals, priorities, and execution signals through generic UI panels, abstract status blocks, signal lines, and a calm central command surface. No real screenshots, no real dashboard data, no inbox, no calendar, no notes, no Gmail, no Drive, no client names, no business names, no personal names, no emails, no phone numbers, no legal material, no family material, no court material, no evidence material, no financial figures, no credentials, no API keys, no repository secrets, no private route labels, no mature SaaS claims, no uptime claims, no security claims, no enterprise-scale claims, no third-party logos, no cluttered UI, no dense readable body text.

Required visual safety controls:

Use staged/fictional data only. Keep visible text minimal and generic. Avoid any implication that this is a live private OS screenshot.

Proposed filename:

`os3-dash-command-dashboard.webp`

Proposed registry key:

`os3DashCommandDashboard`

Proposed alt text:

OS3 Dash command dashboard concept showing structured workflow visibility.

Target aspect ratio:

16:10 preferred, 4:3 acceptable.

Desktop crop guidance:

Show the central command surface with 3-4 clear panels and a visible signal/workflow path.

390px mobile crop guidance:

Crop to one central dashboard plate and avoid tiny text.

Human approval checklist:

- Prompt is public-safe.

- Filename, registry key, alt text, and crop plan approved.
